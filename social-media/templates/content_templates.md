# Geekbobo 內容模板指引

## 寫作原則
所有內容必須遵守以下原則，確保「真實朋友」的感覺：

1. 永遠用第一人稱「我」(Martin)
2. 語氣像 WhatsApp 同朋友傾偈
3. 唔好用任何公關用語（「領先」「創新」「一站式解決方案」）
4. 每篇貼文最多只能有一個輕度 CTA（例如「你覺得呢？」）
5. 唔好每篇都提產品，有時純粹分享感受就夠

---

## 模板 A：Build in Public 更新

```
[今日做咗咩]

簡短描述今日的開發進度或決策。

[點解要做呢個]

用 1-2 句解釋背後的原因，最好連結到真實用戶的需要。

[真實感受]

分享過程中的困難、驚喜或學到的嘢。

[開放式問題]

問一個跟受眾相關的問題，鼓勵留言。
```

**例子**：
「今日終於搞掂咗 Cardlink 嘅多語言支援。因為有個用戶同我講佢啲客人有講普通話嘅，如果介面得英文同廣東話就唔夠。呢個功能比我預期難做好多，但做完之後覺得好有成就感。你哋做生意有冇遇過語言嘅問題？」

---

## 模板 B：創業故事 / 觀察

```
[場景描述]

用 1-2 句設定場景（例如：去咗邊、見咗邊個）。

[觀察 / 發現]

描述你觀察到嘅問題或現象。

[個人反思]

分享呢件事對你嘅影響或啟發。

[連結到受眾]

問一個讓受眾分享自己經歷的問題。
```

**例子**：
「今日去咗深水埗一間文具舖，老闆娘用緊一本手寫簿記錄所有出入貨。我問佢點解唔用電腦，佢話：『我試過，但啲系統太複雜，我學唔識。』呢句說話令我好深刻。科技應該係幫人，唔係嚇人。你身邊有冇人因為覺得科技太複雜而放棄用？」

---

## 模板 C：價值分享 (Tips / Insights)

```
[Hook - 直接講重點]

用一句引起注意的話開頭。

[內容 - 2-3 個 points]

分享你的觀察或建議，但用「我發現」而唔係「你應該」。

[個人經驗]

用自己或身邊人的真實經歷佐證。

[邀請討論]

「你同意嗎？」或「你嘅經驗係點？」
```

---

## 模板 D：互動帖 (純對話)

```
[簡單問題]

問一個低門檻、容易回答的問題。可以係 A/B 選擇、填空、或者分享經歷。
```

**例子**：
- 「做老闆最孤獨嘅時刻係咩？我先講：係凌晨 2 點仲對住個 spreadsheet 嘅時候」
- 「你間舖而家最想改善嘅一樣嘢係咩？」
- 「如果可以重新嚟過，你開舖第一日會做唔同嘅咩？」

## IG Stories 指引
- 永遠用手機直接拍，不加濾鏡
- 可以加簡單文字，但唔好用花巧嘅 sticker
- 內容：工作枱面、電腦螢幕、食飯、搭車、任何真實嘅日常
- 偶爾加 Poll 或 Question sticker 增加互動
